# /investigate

七步调查法，用于深度分析一个主题。
这是Garry Tan Diarization概念的核心实现。

## 描述

执行系统性调查，遵循严格的七步流程：
1. 确定数据范围
2. 构建时间线
3. 编年体摘要(diarize)
4. 综合分析
5. 正反论证
6. 引用来源
7. 得出结论

## 参数

- TARGET: 调查目标（人、公司、事件）
- QUESTION: 核心问题
- DATASET: 数据集路径或源

## 步骤

### 1. Scope数据集

确定可用数据边界：
- 数据来源有哪些？
- 时间范围是什么？
- 数据完整度如何？

### 2. 构建时间线

梳理事件先后顺序：
- 关键节点
- 参与者行动
- 里程碑事件

### 3. 编年体摘要(Diarization)

**这是Garry Tan的核心洞察：读50份文档，产出1页结构化判断**

不是简单的总结，而是：
- 捕捉矛盾点
- 注意到变化
- 区分"说"vs"做"

格式：
```
SAYS: [文档中声称的目标]
ACTUALLY: [实际行为反映的目标]

关键矛盾:
- [矛盾1]
- [矛盾2]
```

### 4. Synthesize

综合所有来源：
- 共同主题
- 分歧点
- 模式识别

### 5. Argue Both Sides

正反两面论证：
- 支持结论的证据
- 反对结论的证据
- 质疑点

### 6. 引用来源

所有结论必须有据可查：
- 来源1: [文件/记录]
- 来源2: [文件/记录]

### 7. 得出结论

基于证据的概率判断，不是确定性断言：
- 高度可能
- 中度可能
- 不确定

## 示例

```
/investigate TARGET="Dr. Sarah Chen" QUESTION="是否在研究中被噤声" DATASET="./emails/"

Same skill, different data:
- → 医学研究分析师
- → 取证调查员
- → 合规审计员
```

## 关键洞察

**Skill File像方法调用**：同一技能，不同参数=不同能力

这不是Prompt Engineering，这是Software Design：
- Markdown作为编程语言
- 人类判断作为运行时
- 参数化实现复用
